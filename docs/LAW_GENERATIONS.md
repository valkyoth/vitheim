# Composite Security Law Generations

Status: normative planning registry introduced by `0.18.3`

This table is the canonical historical contract for every `VIT-LAW-*`.
`docs/INVARIANT_OWNERSHIP.md` shows the latest resolved generation; it does not
retroactively redefine an earlier release. A generation starts from its
predecessor's exact dependency set, applies the explicit additions/removals, and
resolves `unchanged` semantic fields from that predecessor. Generation one must
state the complete linearization, failure, and recovery semantics.

An invariant dependency cannot enter a law before the invariant's lifecycle
effective version. Mixed-version operation is the conservative intersection of
the predecessor and successor generations: a node that cannot validate an added
root cannot exercise the expanded law. Generation activation, migration,
rollback, dependency-set, and recovery-proof contracts are append-only.
